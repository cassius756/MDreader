## 增加数学公式解析能力 — 实施计划

### 目标
为 MD 阅读器增加 LaTeX 数学公式渲染,支持行内公式 `$...$`、`\(...\)` 和块级公式 `$$...$$`、`\[...\]`。**完全遵循现有 mermaid 的集成模式**(本地优先→CDN 回退、后处理函数、try/catch 容错),保持单文件自包含架构。

### 技术选型
**KaTeX** + auto-render 扩展(已确认)。理由:渲染快、标准、与现有 mermaid 模式一致。需要加载 3 个文件(JS+CSS+auto-render)。

---

### 改动点(全部在 `code.html`,共 4 处)

#### ① 新增 `loadStyle` 辅助函数(第 614-622 行 `loadScript` 之后)
KaTeX 需要 CSS,现有 `loadScript` 只加载 JS。新增一个 Promise 化的 `<link>` 注入器,与 `loadScript` 并列:
```js
function loadStyle(href) {
  return new Promise((resolve, reject) => {
    const l = document.createElement('link');
    l.rel = 'stylesheet';
    l.href = href;
    l.onload = resolve;
    l.onerror = reject;
    document.head.appendChild(l);
  });
}
```

#### ② 新增 `loadKatex` 加载 IIFE(第 643 行 `loadMermaid` 之后)
镜像 `loadMarked`/`loadMermaid` 的本地优先→CDN 回退结构。加载顺序:CSS → 核心 JS → auto-render JS。
- 本地:`katex.min.css` / `katex.min.js` / `auto-render.min.js`(若用户放置)
- CDN:`https://cdn.jsdelivr.net/npm/katex@0.16.11/dist/...`
- 设置 `katexReady` 标志(类比 `markedReady`),失败静默降级(公式以原始文本显示,不报错)。

#### ③ 新增 `renderMath()` 函数(第 719 行 `renderMermaidDiagrams` 之后)
镜像 `renderMermaidDiagrams` 的结构:守卫→处理→try/catch。调用 auto-render 的 `renderMathInElement`:
```js
function renderMath() {
  if (!katexReady || typeof renderMathInElement === 'undefined') return;
  try {
    renderMathInElement(contentArea, {
      delimiters: [
        { left: '$$', right: '$$', display: true },
        { left: '\\[', right: '\\]', display: true },
        { left: '$', right: '$', display: false },
        { left: '\\(', right: '\\)', display: false }
      ],
      throwOnError: false,
      ignoredTags: ['script', 'noscript', 'style', 'textarea', 'pre', 'code']
    });
  } catch (e) {
    console.warn('公式渲染失败:', e);
  }
}
```
关键细节:
- `throwOnError: false` — 公式语法错误时显示原错误信息而非崩溃(对阅读器很重要)。
- `ignoredTags` 含 `pre`/`code` — **避免在代码块内的 `$` 被误判为公式**(这是常见的坑)。
- 必须在 mermaid 之后渲染(mermaid 会把 `language-mermaid` 的 `<pre>` 换成 `<div>`,此时数学公式才不会受影响)。

#### ④ 在 `renderFile` 中调用(第 663 行 `renderMermaidDiagrams();` 之后新增一行)
```js
contentArea.innerHTML = marked.parse(text);
renderMermaidDiagrams();
renderMath();              // ← 新增
```
**只在 marked 分支调用**(`simpleMarkdown` 回退分支不加,与 mermaid 保持一致 — 回退场景下 CDN 通常也不可用)。

---

### 关键设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 解析方式 | auto-render 后处理(非 marked 扩展) | 完全复用 mermaid 的成熟模式,不侵入 marked 配置,零回归风险 |
| `$...$` 行内分隔符 | 启用 | 用户需求;`ignoredTags` 保护代码块避免误判 |
| 回退分支 | 不支持公式 | 与 mermaid 一致;CDN 不可用时 KaTeX 同样不可用 |
| `.txt` 纯文本分支 | 不渲染 | 正确,txt 非 markdown |
| 容错 | `throwOnError: false` + try/catch | 单个公式出错不影响其他内容和文件切换 |

### 验证方式
准备一个测试 `.md` 文件,包含:
- 行内:`$E=mc^2$` 和 `\(\alpha+\beta\)`
- 块级:`$$\int_0^\infty e^{-x} dx$$` 和 `\[\sum_{i=1}^n i\]`
- 代码块内含 `$` 符号(验证不被误渲染)
- 故意写一个语法错误的公式(验证 `throwOnError: false` 生效)

分别在「联网 + CDN」「本地放置 katex 文件」「断网」三种场景测试。
