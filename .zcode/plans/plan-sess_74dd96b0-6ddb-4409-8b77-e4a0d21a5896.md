## "最近打开"功能 — 实施计划

### 用户决策（已确认）
- **重打开方式**:真正自动重打开 → 采用 File System Access API + IndexedDB 持久化 handle
- **UI 位置**:侧栏底部 + 欢迎页

---

### 核心架构

#### 浏览器能力分层（关键）
| 能力 | 支持 | 用途 |
|------|------|------|
| `'showDirectoryPicker' in window` | Chrome/Edge ✓,Firefox/Safari ✗ | 真正一键重打开 |
| `FileSystemHandle` + IndexedDB | Chrome/Edge ✓ | handle 跨刷新持久化 |
| `<input webkitdirectory>`(现有) | 全部 ✓ | 降级时的原生选择器 |

**降级策略**:
- 支持 FS Access API → 点击最近项直接 `handle.requestPermission()` + 重新遍历,一键直达
- 不支持 → 显示"请重新选择"toast + 自动触发原生 `<input>`,用户手动确认
- 检测结果以 `fsAccessSupported` 常量统一缓存

#### 数据模型
统一记录(文件夹和单文件用同一结构,用 `kind` 区分):
```js
// IndexedDB store: 'recent', keyPath 'id'
{
  id: string,              // 唯一标识:kind + '|' + name(去重用)
  kind: 'directory' | 'file',
  name: string,            // 显示名(文件夹名/文件名)
  handle: FileSystemHandle | null,   // FS Access API handle(降级时为 null)
  fileCount: number,       // 文件夹专属,文件数为 1
  openedAt: number         // Date.now(),用于排序
}
```
- **去重**:同一文件夹重复打开只保留一条,更新 `openedAt`
- **容量**:最多 10 条,超出删最旧
- **排序**:按 `openedAt` 倒序

#### ⑤ 重打开逻辑(新增函数 `reopenRecent(entry)`)
```js
async function reopenRecent(entry) {
  if (entry.kind === 'directory' && entry.handle) {
    // FS Access 路径
    const perm = await entry.handle.queryPermission({ mode: 'read' });
    if (perm !== 'granted') {
      const req = await entry.handle.requestPermission({ mode: 'read' });
      if (req !== 'granted') { showToast('已拒绝访问权限'); return; }
    }
    try {
      const files = await readDirViaHandle(entry.handle);
      if (files.length === 0) { showToast('文件夹中没有 .md 或 .txt 文件'); return; }
      window.__lastDirHandle = entry.handle;
      folderFiles = files.sort((a,b) => a.path.localeCompare(b.path));
      finishOpenFolder();
      showToast('已打开:' + entry.name);
    } catch (e) { showToast('打开失败,可能已移动或删除'); }
  } else {
    // 降级路径(无 handle 或不支持)
    showToast('请重新选择"' + entry.name + '"');
    (entry.kind === 'directory' ? folderInput : fileInput).click();
  }
}
```

### 关键设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 存储 | IndexedDB | 唯一能存 `FileSystemHandle` 的存储;localStorage 不支持 handle 序列化 |
| handle 可空 | ✓ | 兼容 drop/input 流,这些路径无 handle 但仍记录名称 |
| 文件夹/文件统一 | 同一 store、同一结构 | 简化数据模型,UI 用图标区分 |
| 降级 | 自动检测 + toast 提示 | Firefox/Safari 用户仍能用,只是多一步手动确认 |
| 不破坏现有快捷键 | Ctrl+O 仍走 input | 避免改快捷键记忆;FS Access 由工具栏按钮触发 |
| 权限复用 | queryPermission→requestPermission | 已授权过的不重复弹窗 |

---

### 边界情况

| 场景 | 行为 |
|------|------|
| 文件夹被删除/移动 | requestPermission 成功但遍历失败 → toast "打开失败" |
| 权限被拒 | toast "已拒绝访问权限" |
| 同名文件夹再次打开 | 去重,只更新时间戳 |
| 超过 10 条 | 删除最旧 |
| Firefox 打开 | 全程走 input 路径,记录名称无 handle,重打开时提示重选 |
| IndexedDB 被禁 | 静默失败,功能不可用但应用正常 |
| 重打开历史里的文件夹,文件已被改动 | 重新遍历读取最新内容 ✓ |
